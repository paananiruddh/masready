MADD Prompt Pack v0.1

10 Maximo troubleshooting prompts for humans using AI responsibly. Redact sensitive data before using any AI tool.

1. Automation Script Failure

For BMX errors, launch points, MBO lifecycle issues, and script failures.

I am troubleshooting an IBM Maximo automation script issue. Context: Maximo version, environment, object, script name, launch point type, error message, recent changes, and what I checked. Do not suggest update/delete SQL. Ask for missing evidence. Separate likely root causes, read-only checks, test plan, and rollback notes.

2. Escalation Not Firing

Help me diagnose a Maximo escalation that is not firing. Ask for escalation status, condition, schedule, reference point, site/org scope, cron status, where clause, and recent changes. Provide read-only validation checks and likely causes.

3. Cron Task Stuck

Help me troubleshoot a Maximo cron task that is stuck or not processing. Ask for cron task name, instance, schedule, parameters, server, logs, queue depth, and recent deployments. Provide safe checks only.

4. MIF / Integration Error

Help me diagnose a Maximo integration issue. Ask for object structure, endpoint, external system, enterprise service/publish channel, error text, payload shape, and processing state. Produce likely root causes and read-only checks.

5. Security Access Mystery

Help me reason through a Maximo access issue. Ask for user, groups, application, signature option, site/org, conditional UI/security, and expected access. Do not recommend broad admin access. Provide least-privilege analysis.

6. Environment Drift

Compare a Maximo issue between DEV, TEST, and PROD. Ask for configuration objects, scripts, escalations, cron, domains, object structures, and versions. Identify drift hypotheses and validation checks.

7. Patch / iFix Impact

Help assess potential Maximo patch/iFix impact. Ask for release notes, current version, industry solutions, custom scripts, integrations, cron/escalations, and heavily used applications. Produce impact areas and retest scope.

8. MAS 9 Migration Readiness

Help structure a MAS 9 migration readiness review. Ask for current version, customisations, integrations, authentication, reports, automation scripts, JVM/logging patterns, and data volumes. Produce risk register and workstream plan.

9. License Usage Review

Help analyse Maximo usage signals responsibly. Ask for logintracking, groups, apps, roles, business process, and usage period. Do not make legal licensing claims. Provide planning insights and questions for IBM/licensing owner.

10. Root Cause Solution Pack

Turn this troubleshooting case into a solution pack. Include summary, evidence, assumptions, likely root cause, read-only validation checks, fix options, risks, test plan, rollback notes, and human approval checklist.

Responsible use note

This pack is not production advice. It is a thinking aid for Maximo practitioners using AI responsibly. Remove customer names, URLs, usernames, API keys, passwords, tokens, DB strings, logs with identifiers, and confidential data before using any prompt.
