# Product Overview

## MASReady — Maximo Delivery Automation Workbench

MASReady is a delivery intelligence and automation platform purpose-built for IBM Maximo Application Suite (MAS) implementation teams.

## Core Value Proposition

Enterprise IBM Maximo implementations are complex, high-risk, and expensive. MASReady reduces delivery risk by combining disjointed Maximo delivery processes — requirements, environment scanning, patch analysis, licence planning, skill mapping, and trust controls — into a single evidence-backed workbench.

## Six Core Modules

| Module | What it does |
|--------|-------------|
| Delivery Intelligence | Unified confidence score aggregating all delivery dimensions |
| Maximo Fingerprint | Automated scan of installed objects, customisations, version data |
| Patch Impact Analysis | Cross-references patch manifests against environment fingerprint |
| License Planning | AppPoint trends, named users, mobile pool analysis |
| Skill Coverage | Maps team capabilities against requirements and patch impact |
| Trust Center | Audit-first read-only design with immutable trace capture |

## Trust Model (Critical)

MASReady operates on a zero-mutation principle:
- No SQL execution against production systems
- No Jira write operations
- No Maximo write operations
- All integrations are read-only
- Complete immutable audit trail of every action
- Trust boundary = REVIEW ONLY

## Demo Tenant

The **MAS9 Power** fictional tenant is an Energy & Utilities company running IBM MAS Manage 9.x. It demonstrates all six modules with realistic but entirely fictional data.

## Target Personas

| Role | Use of MASReady |
|------|----------------|
| Admin | System configuration, trust boundary management, audit |
| Delivery Lead | Intelligence score, requirements, patch impact, skill gaps |
| Asset Manager | Maximo fingerprint, customisation inventory |
| Finance | License AppPoint planning, utilisation reporting |
| Auditor | Immutable audit log, trust confirmation |
| Field Tech | (Demo only) Work order and asset data context |
| Scheduler | (Demo only) PM and job plan context |

## Differentiators

- Zero production mutations — proven, not claimed
- Evidence-backed outputs — every finding has a source trace
- Review-ready — outputs go directly into client review packs
- Role-aware — each persona sees relevant context only
- Offline-capable — skill packs and guides work without internet during workshops
