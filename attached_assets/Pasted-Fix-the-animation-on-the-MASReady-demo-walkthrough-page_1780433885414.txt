## Main issue

This is not a design preference issue.

The page says/implies animation, but the page does not actually animate.

The animation system is either:

* not wired
* not firing
* being blocked
* too subtle to see
* broken by layout/CSS
* broken by client/server rendering
* broken by scroll container handling

Fix the actual execution problem.

The page must visibly animate.

Users should clearly see:

* hero entrance animation
* section reveal animation
* staggered cards
* KPI counters/count-up animation
* walkthrough stages animating as the user scrolls
* smooth transitions between demo steps
* tab/stage transitions where relevant
* dashboard/demo cards feeling alive

This must not be only hover effects or static CSS.

## Find why animation is not running

Specifically check for these failure points:

### Framer Motion

Verify:

* `framer-motion` is installed
* components use `motion.div` or equivalent
* `initial` values are set
* `animate` or `whileInView` values are set
* `transition` values are meaningful
* `viewport` settings are not preventing triggers
* components are client-rendered where required
* no hydration mismatch is disabling animation
* variants are actually passed into components
* child components are not plain `div`s when they should be `motion.div`s

Correct pattern example:

```tsx
<motion.div
  initial={{ opacity: 0, y: 48 }}
  whileInView={{ opacity: 1, y: 0 }}
  viewport={{ once: true, amount: 0.2 }}
  transition={{ duration: 0.65, ease: "easeOut" }}
>
  ...
</motion.div>
```

### GSAP / ScrollTrigger

If GSAP is used, verify:

```ts
gsap.registerPlugin(ScrollTrigger)
```

Also verify:

* ScrollTrigger is imported correctly
* animations are created after DOM is mounted
* cleanup is handled
* refs point to actual DOM nodes
* ScrollTrigger uses the correct scroll container
* no route wrapper prevents scroll detection
* pinned sections are not broken
* trigger elements exist
* start/end positions are valid

### CSS / Tailwind blocking

Check for:

* `opacity-0` never removed
* `transform-none` overriding animated transform
* `transition-none`
* `overflow-hidden` clipping animated content
* parent containers blocking scroll triggers
* `will-change` problems
* z-index making animated elements invisible
* fixed/sticky elements covering content
* classes fighting Framer/GSAP inline styles

### Reduced motion

Check:

* `prefers-reduced-motion`
* custom reduced motion hooks
* browser/system reduced animation settings
* any global flag disabling animation

Important:

Reduced motion support is fine, but it must not accidentally disable animation for everyone.

### Client-side rendering

If this is Next/Vite/React with SSR-style constraints, check:

* page/component has `"use client"` if needed
* animation code does not run on server
* no `window` access before mount
* no hydration mismatch
* no dynamic import timing issue

## Motion quality rules

Do not add childish/gimmicky motion.

Avoid:

* bouncing
* spinning
* excessive zooming
* random movement
* distracting loops
* motion that slows usability

Use:

* premium SaaS motion
* subtle easing
* clear stagger
* smooth section reveal
* operational dashboard-style animation

## Performance rules

* Avoid heavy animation on every frame.
* Do not animate layout-heavy properties unnecessarily.
* Prefer transform/opacity.
* Do not create memory leaks with ScrollTrigger/listeners.
* Respect reduced motion properly, but do not globally disable animation by mistake.
