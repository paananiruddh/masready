COMMENT REPLIES
============================================================

Create suggested replies to likely LinkedIn comments:

1. “Is this just another chatbot?”
Answer:
No — the direction is deliberately Maximo-specific and skill-pack-first, not a generic chatbot wrapper.

2. “Does it connect to live Maximo?”
Answer:
The public simulator does not. Customer-specific Azure-hosted demos can be configured with approved integrations and data boundaries.

Answer:
No. The current model is review-only: no SQL execution, no Maximo mutation, no Jira/ADO mutation.

Answer:
Demo fixtures are demo-only. Real customers choose a data model: no-store, customer-hosted, BYO storage, Azure-hosted customer subdomain, or controlled managed demo workspace.

5. “Why local skill packs?”
Answer:
Because Maximo delivery depends on domain-specific patterns, version context, customisation evidence, and governance — not just generic text generation.

6. “What is next?”
Answer:
More Maximo skills, more industry scenarios, skill-quality scoring, patch/iFix insights, customer-derived benchmarks, and more Azure-hosted customer demos.
