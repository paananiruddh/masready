============================================================
CORE PROBLEM
============================================================

The current flow is still broken.

Observed issue:

- User clicks “Get MASReady demo link” or similar CTA.
- A form opens.
- User fills the form.
- They get a thank-you message.
- But then what?
- They do not get a UUID URL.
- They do not get a new window.
- They do not get an actual synthetic preview session.
- It still feels like form → form → form → thank you → nothing.

This must stop.

A user who requests a public synthetic demo link must immediately receive a usable, time-limited UUID-based preview URL.

The form should not be called “Get MASReady Demo Link” if it does not actually generate and open a link.

============================================================

For public synthetic preview/demo-link flow:

On form submission or wizard completion:

1. Generate a UUID-based preview session.
2. Build a URL:
   /preview-session/<uuid>
3. Show the URL clearly on the current page.
4. Automatically open the URL in a new browser tab/window.
5. Also provide a button:
   Open Preview Link
6. Also provide a copy button:
   Copy Preview Link
7. Show thank-you/confirmation on the current page only as secondary information.

The user must never be left with only “Thank you” and no URL.
