# Media Safety Guidelines

## What Must NOT Be Published

- Playwright trace zip files
- Raw DOM traces or debug screenshots
- Secrets, tokens, or API keys
- Real customer data of any kind
- Localhost URLs or internal debug output
- Raw credentials or unreviewed content

## What Is Safe to Publish

- Fictional seeded demo metrics (clearly labelled as demo data)
- Generated or recreated SVG charts using fictional data
- Mermaid architecture diagrams
- Public documentation summaries
- Safety disclaimers
- Static simulations using fictional fixtures

## Required Disclaimers

Every page showing demo data must include one of:
- "MAS9 Power is fictional"
- "Static guided simulation — no real customer systems connected"
- "Fictional demo data only"
- "Review-only output"
