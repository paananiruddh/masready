New Feature

Adaptive Regression Intelligence

Your Maximo environment changes. Your regression suite should change with it.

MASReady turns the Maximo environment fingerprint into a customer-specific Playwright regression pack. Regression coverage grows or shrinks as installed components, add-ons, integrations, and customisations change — all with human review before any test reaches a pipeline.

How it works

Seven stages from Maximo fingerprint to CI/CD-ready Playwright suite — with mandatory human review before any generated test is used.

  1. 01
    Maximo Fingerprint Captured — Read-only scan of installed applications, add-ons, automation scripts, workflows, escalations, integrations, cron tasks, security groups, and UI customisations.
  2. 02
    Component & Change Detection — MASReady identifies what is installed, what has changed since the last fingerprint, and what has been removed — at the component level.
  3. 03
    Regression Scope Analysis — Each detected component, customisation, integration, and workflow is mapped to its test risk profile. High-risk areas drive higher test density.
  4. 04
    Playwright Test Pack Generated — MASReady produces customer-specific TypeScript Playwright tests grouped by Maximo domain and risk tier. No generic scripts — everything reflects the actual environment.
  5. 05
    Human Review & Approval — Generated tests are presented as recommendations only. QA engineers review the scope, modify as needed, and approve before any test is used in a pipeline.
  6. 06
    CI/CD Handoff — Approved tests are exported as standard TypeScript Playwright files with a pipeline guide for GitHub Actions, Azure Pipelines, Jenkins, or GitLab CI.
  7. 07
    Drift-Triggered Updates — When significant environment drift is detected — new add-on, removed workflow, changed integration — MASReady recommends targeted test additions, removals, or updates.

Architecture & trust boundaries

Adaptive Regression Intelligence inherits MASReady's core trust model. Fingerprinting is always read-only. Generated tests are always customer-controlled.

Read-only fingerprinting

  • Maximo API access uses read-only credentials only
  • Zero mutations to source system at any stage
  • Fingerprint reads environment state — it never writes
  • No data written to Maximo during scanning

Human review gate

  • Generated tests are recommendations, not automations
  • QA engineers review all proposed test additions
  • No test reaches a pipeline without explicit approval
  • Rejection or modification is always an option

Customer-controlled export

  • Exported test files are plain TypeScript — no lock-in
  • Customer runs tests in their own pipeline
  • MASReady does not execute tests on customer systems
  • Export includes full source — nothing is hidden

Data boundary

  • Fingerprint data stays within the customer data boundary
  • No environment data shared between tenants
  • Generated tests reference no proprietary environment data
  • Customer chooses data mode (hosted, zero-retention, BYO)

Frequently asked questions

What is Adaptive Regression Intelligence?

A MASReady feature that uses the Maximo environment fingerprint to generate, maintain, and recommend a customer-specific Playwright regression suite — reflecting the actual installed components, not a generic Maximo template.

Does MASReady run tests against production?

No. MASReady generates test files for customer use. Tests are exported and run by the customer in their own pipeline. MASReady never executes tests against customer systems.

Does it replace QA teams?

No. MASReady generates a starting point and a recommended scope. Human QA review and approval is required before the suite is used. QA teams own the decision to accept, modify, or reject generated tests.

How does MASReady know which tests to generate?

From the environment fingerprint — installed applications, add-ons, automation scripts, workflows, escalations, integrations, cron tasks, security groups, application designer changes, and UI customisations.

What happens when the Maximo environment changes?

MASReady detects drift between the captured fingerprint and the current environment. When significant changes are found, it recommends test additions, removals, or updates — which human QA reviews before adoption.

Can customers export the Playwright suite?

Yes. The generated test pack is fully exportable as TypeScript Playwright files, ready for the customer's own CI/CD pipeline. MASReady does not retain or run these files.

Can it work with CI/CD?

Yes. Exported tests run in standard CI/CD environments (GitHub Actions, Azure Pipelines, Jenkins, GitLab CI). A CI/CD handoff guide with pipeline configuration is included with every export.

Does it mutate Maximo?

No. Fingerprinting uses read-only APIs exclusively. MASReady cannot write to, modify, or trigger any action in Maximo or any connected system.
